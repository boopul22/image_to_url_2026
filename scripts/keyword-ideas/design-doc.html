1. Overview

ImageToURL operates imagetourl.cloud, a free online tool that converts an uploaded image into a hosted direct URL. The site is monetized through display advertising and does not sell a product. We are building a small internal command-line script to perform keyword research that informs the SEO content and landing pages we publish on our own website. The tool is used only by the site owner; it is not a product, is not distributed, and is not made available to any third party or the public.

2. Google Ads API usage

  • Service used: KeywordPlanIdeaService
  • Method used: GenerateKeywordIdeas (read-only)
  • Seeds: a short list of seed keywords and our own site URL (imagetourl.cloud).
  • Targeting: language = English, geo = United States (additional locales later).
  • No mutate/write calls. The tool only reads keyword ideas and metrics (average monthly searches, competition, and top-of-page bid ranges). It does not create, edit, pause, or delete campaigns, ads, budgets, or any account entity.

3. Architecture

A Node.js (v22) script using the open-source google-ads-api client library.

  1. OAuth2: a one-time consent flow mints a refresh token with the https://www.googleapis.com/auth/adwords scope. Credentials are stored in a local, git-ignored environment file.
  2. The script authenticates with the developer token (manager account 271-257-4999) plus the refresh token, and queries GenerateKeywordIdeas against our own Google Ads account (205-600-6470).
  3. Results are written locally to CSV and JSON files and printed as a console table for manual review during content planning.

4. Users and access

Internal only — the single site owner runs the script from a local machine. There is no user interface, no login system, no multi-tenant access, and no redistribution of API data.

5. Request volume and quotas

Very low. The script is run manually, on demand, a handful of times per week, with a small number of seed keywords per run. We will respect all rate limits and use-quota restrictions and will not attempt automated high-frequency querying.

6. Data handling and compliance

Keyword-idea data is used solely for our own internal SEO planning and stored locally. It is not shared, resold, or exposed to end users. We will keep the API contact email current and comply with the Google Ads API Terms and Conditions and all applicable policies.
