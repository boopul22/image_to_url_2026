🔗 "url" vs "link" — two separate keyword families

"X to url" and "X to link" are distinct queries — separate impressions, separate rankings, separate intent in Google's eyes. Both are relevant and both deserve to be targeted. Having "link" on a page is a baseline, not a finish line: a page can rank for "jpg to url" yet still convert badly on "jpg to link". So no page here is "done" — every one has a live "link" opportunity, even the ones that already use the word.

🕸️ Internal linking — the real bottleneck

This is what the GSC data couldn't show but the code makes obvious. It explains why high-intent pages sit on page 2 despite big impressions.

✗ Homepage body links to ZERO landing pages. The only hrefs in HomePage.astro are 4× /api/auth/login. Your highest-authority page passes none of that authority to jpg/png/photo/converter pages.
✗ No sibling cross-linking. The landing template ([...slug].astro) has no relatedTools field. All ~28 pages link only back to the homepage — equity flows in, never sideways.
~ Nav/footer links exist but are weak. High-value pages are in nav-links.ts, but rendered inside JS-collapsed dropdowns / <details> accordions — boilerplate sitewide links pass thin, undifferentiated equity.
The fix is high-leverage and low-effort: (1) add a contextual "Popular tools" link grid in the homepage body (real <a> links) pointing at the page-2 keyword pages; (2) add a relatedTools section to the landing template so siblings interlink. Both are pure internal-link wins that directly push "image to url", "jpg to url", "image to url converter" upward — no backlinks required.
